# ⚡ 电力数据资产管理平台

> 面向电力设计院场景的数据资产全生命周期管理 Demo：从多源数据接入、治理、编目和产品上架，到门户申请与审批。

本项目用于**数据治理 AI 工程师岗位面试演示**，重点展示需求理解、产品拆解、功能矩阵、业务流程和前端交互。当前所有业务数据均为 Mock 数据，任务执行、状态变化和审批结果主要由 React 本地状态或轻量会话状态模拟，不连接真实后端。

## 🎯 项目背景与平台定位

电力设计院的数据来源通常覆盖业务数据库、外部 API、勘测与 GIS 文件、设计成果、工程文档和实时监测数据。不同数据在格式、空间参考、质量、安全等级和使用方式上存在差异，适合用来展示数据逐步资产化、产品化的完整过程。

本 Demo 将平台定位为连接数据建设端与使用端的统一入口：

```text
外部数据 → 标准化数据 → 数据资产 → 数据产品 → 可申请使用的数据服务
```

项目业务聚焦广东、广西、海南等华南地区，主要业务域包括技经数据、勘测数据、电气设计数据、工程数据和标准知识库。项目中的组织、工程、人员和运营数据均为模拟内容，不代表真实企业结论。

## 🧭 需求理解

面试题的原始目标是：**从外部数据接入，到形成数据资产目录，最终在门户上架并提供使用。**

项目将其拆解为以下产品链路：

| 环节 | 解决的问题 |
| --- | --- |
| 数据源 | 明确数据库、API、文档、GIS、设计成果和实时数据等来源 |
| 数据接入 | 统一配置连接、预览结构、设置同步策略并追踪执行记录 |
| 数据治理 | 处理标准、质量、元数据、分类分级与安全等级问题 |
| 数据资产目录 | 将治理成果组织为可检索、可评价、可管理的数据资产 |
| 数据产品上架 | 对已申请上架的资产进行产品封装、质量安全检查与权限配置 |
| 数据产品门户 | 面向使用者提供产品搜索、筛选、详情、收藏和申请入口 |
| 申请与审批 | 在个人工作台集中管理我的申请、待办审批和经手数据 |

## 🔄 核心业务流程

```mermaid
flowchart LR
    A[外部数据源] --> B[数据接入]
    B --> C[数据治理]
    C --> D[数据资产目录]
    D --> E[申请上架]
    E --> F[数据产品上架]
    F --> G[数据产品门户]
    G --> H[用户申请]
    H --> I[审批与授权]
```

## 各模块详细说明

### 登录认证

- 演示账号采用固定用户名和密码。
- 未登录访问业务路由会重定向到 `/login`；登录后刷新页面仍可访问。
- 登录标识保存在 `sessionStorage`，不保存明文密码。
- Sider 底部用户菜单可进入个人工作台或确认退出登录。
- 该实现仅用于本地 Demo。生产环境需要后端认证、密码哈希、Token、会话失效和服务端权限校验。

### 数据资产概览

- 展示数据源、数据资产、数据产品、API 服务、质量评分和访问量等核心指标。
- 展示五类业务域的资产分布、最新动态和热门资产。
- 将数据产品门户、个人工作台作为核心入口。
- 以 1—4 的顺序展示数据接入、数据治理、数据资产目录和数据产品上架，并支持点击跳转。

### 数据产品门户

- 支持按名称、分类、标签、地区、来源和场景搜索。
- 支持业务领域、数据形态、安全等级、地区筛选及推荐、时间、申请量、质量排序。
- 页面采用紧凑搜索与统计头部，产品列表会根据桌面宽度以四列、三列或两列自适应展示，便于连续浏览更多产品。
- 产品详情包含基本信息、资源内容、质量更新和使用说明。
- 产品卡片支持收藏或取消收藏；申请入口位于详情 Drawer。
- 申请表自动带入当前模拟用户“张数据”和“数据管理中心”，提交后同一会话内可跳转工作台查看对应申请。
- 搜索、收藏和申请状态均为前端演示状态，刷新后可能恢复。

### 数据接入

- 接入能力按“接入模式 → 数据源类型 → 数据格式”组织：连接接入包含数据库连接和 API 接口，文件导入包含通用文档、GIS 与勘测成果和设计成果，实时流接入包含实时数据流。
- 六类数据源入口在常见桌面宽度下保持单行排列，并通过分类标题、短横线和细分隔线弱化分组，兼顾分类语义与页面紧凑度。
- 新建流程包含选择数据源、配置连接、预览与识别、同步配置四步，以及模拟执行结果。
- 配置会根据数据源类型动态切换；文档编码支持自动识别、UTF-8、GBK 等选项。
- 已接入数据源支持详情、编辑、同步、停用或启用。
- 接入执行记录支持查看日志；失败或部分成功任务可模拟重试。
- 文件上传、连接测试、字段识别和任务执行均为模拟，不会访问真实数据源。

### 数据治理

- 治理任务覆盖待治理、治理中、待确认、已完成和失败等状态。
- 同一治理入口会根据状态执行开始治理、继续治理或重新治理。
- 工作区包含标准化处理、数据质量检查、元数据完善、分类分级与标签、治理确认五步。
- 质量问题支持单条处理及全部接受、全部复核、全部忽略。
- 支持保存并退出，并在同一页面会话内恢复草稿步骤。
- 治理完成后可提交数据资产目录；分类与目录页共用三级分类配置。

### 数据资产目录

- 使用可展开的目录筛选面板和三级分类树过滤资产列表。
- 资产详情包含基本信息、元数据、数据质量、数据血缘、分类标签、价值与健康度、使用与运营七个 Tab。
- 支持查看/编辑、调整目录、添加或删除人工标签。
- 资产状态为“治理异常”“待申请上架”“已申请上架”。
- 待申请上架资产可执行上架前检查并提交申请；已申请上架资产可取消申请；治理异常资产可退回治理。
- 不包含已移除的启用、停用或待启用流程。

### 数据产品上架

- 只读取资产目录中处于“已申请上架”状态、且尚未形成当前列表产品的资产。
- 上架工作区包含选择资产、产品信息配置、质量检查、安全检查、权限配置、上架确认六步。
- 支持门户效果预览，并在完成后生成模拟产品编号。
- 已上架产品支持查看和下架；下架后来源资产回到待审核上架列表。
- 质量、安全和权限检查均为规则化 Mock，不包含真实审批引擎。

### 个人工作台

- “我的收藏”支持查看详情，并可在列表或详情中取消收藏。
- “我的申请”支持查看详情；审批中申请可撤回，驳回、过期或草稿申请可修改后重新提交。
- “待我审批”支持查看申请、风险提示，并模拟通过、驳回、要求补充材料和转交。
- “经手数据”展示负责或参与的数据资产、数据产品和审批记录。
- 门户新提交的申请可在同一 SPA 会话内进入“我的申请”并自动定位。
- 待审批列表为独立预置 Mock；它不会自动接收刚从门户提交的同一条申请。

## 典型数据案例：广东省输电线路路径 GIS 数据

该案例在现有 Mock 中具有最完整的语义覆盖：

1. **接入**：`广东输电线路 GIS 数据库`，来源于输电线路数字化设计平台。
2. **治理**：`广东输电线路 GIS 数据治理`，包含坐标、行政区划、高程和字段规范检查。
3. **资产**：`广东省输电线路路径 GIS 数据集`，目录为“电气设计数据 / 线路设计 / 线路路径”。
4. **上架**：该资产以“已申请上架”的预置状态进入待审核上架列表，可执行产品封装。
5. **门户**：同名产品可查看图层、坐标系、质量和使用限制。
6. **申请**：用户可申请文件下载或在线查看，并进入个人工作台。
7. **审批**：管理员可处理同类产品的预置待审批申请。

> 当前 Demo 使用同名或语义一致的预置 Mock 表示不同生命周期阶段。门户申请到“我的申请”、资产申请到上架页存在轻量会话联动；新建接入不会实时生成治理任务，新提交治理资产不会自动进入上架页，门户申请也不会自动进入管理员待审批队列。

## 🛠️ 技术栈

- React 19（JavaScript）
- Vite 8
- Ant Design 6 与 `@ant-design/icons`
- React Router 7
- 本地 Mock 数据与 React Hooks
- `sessionStorage`：保存登录会话标识
- Git / GitHub：分支、Pull Request、Squash Merge 与版本标签
- Codex：辅助代码实现、检查与迭代
- Oxlint：静态检查

项目未使用 TypeScript、Redux、真实数据库、后端 API 或额外图表框架。

## AI 辅助开发方法

项目采用分阶段、可验证的 AI 辅助开发方式：

1. 使用 ChatGPT 辅助场景调研、需求拆解和产品流程设计。
2. 将页面与状态逻辑拆分为边界清晰的 Prompt。
3. 比较技术方案后固定 React、Vite、Ant Design 和 Mock 数据方案。
4. 使用 Codex 实现页面、交互、Mock 数据和局部重构。
5. 通过浏览器检查、lint 和 build 验证关键路径。
6. 使用 Git 功能分支、Pull Request、Squash Merge 和版本标签管理迭代。

AI 用于辅助分析、生成和迭代；产品逻辑、范围取舍、风险判断和最终验收由开发者完成。

## ⚠️ 当前边界与未来扩展

### 当前边界

- 登录认证为前端 Mock，没有真实 Token 和服务端权限校验。
- 没有真实后端、数据库、数据湖或对象存储。
- 不执行真实数据库连接、API 调用、文件上传或调度任务。
- 治理规则、质量评分、分类推荐和执行进度均为模拟。
- 多数业务状态保存在组件本地；刷新页面后可能恢复初始 Mock。
- 部分跨页面状态使用内存 Map 或数组，仅在当前 SPA 会话有效。
- 审批、授权、下载和 API 凭证为演示性入口。

### 未来可扩展

- 企业统一身份认证与组织权限同步
- 后端接口、数据库和状态持久化
- 数据接入调度与失败补偿
- 可配置的数据质量规则引擎
- 元数据自动抽取、语义识别与 AI 辅助编目
- 可计算的数据血缘与影响分析
- 审批流引擎、细粒度权限和审计日志
- AI 辅助质量问题定位与修复建议
