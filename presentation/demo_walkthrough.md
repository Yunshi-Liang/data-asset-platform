# 面试现场 Demo 演示脚本

## 演示准备

- 启动命令：`npm install` 后执行 `npm run dev`
- Windows PowerShell：可使用 `npm.cmd run dev`
- 浏览器建议：Chrome 或 Edge 最新版
- 演示账号：`DataZhang` / `123456`
- 建议窗口尺寸：1440 × 900 或更高，浏览器缩放 100%
- 是否需要提前清除 sessionStorage：建议清除 `data-asset-demo-auth`，确保从登录页开始
- 是否需要刷新恢复初始 Mock 状态：建议演示前刷新一次；刷新会重置大多数业务本地状态
- 预计总演示时间：精简版约 5 分钟，完整版约 8～10 分钟

演示前执行：

```bash
npm run lint
npm run build
npm run dev
```

建议提前打开 `/login`，关闭开发者工具和无关浏览器标签。不要提前点击上架、审批、退回或下架等会改变本地状态的操作。

## 主线案例

- 数据名称：广东省输电线路路径 GIS 数据集
- 上游数据源：广东输电线路 GIS 数据库
- 数据类型：GIS 数据
- 业务场景：输电线路路径比选、交叉跨越分析和工程前期踏勘
- 选择原因：电力设计院特征明显，并在接入、治理、资产目录、上架、门户、申请和审批 Mock 中均有同名或语义一致的数据

> 真实性说明：当前 Demo 通过预置 Mock 展示同一类数据在不同生命周期阶段的状态。资产目录的上架申请可在同一 SPA 会话中被上架页读取，门户新申请可进入“我的申请”；其他环节主要是名称和业务语义连续，并非真实后端驱动的同一条记录实时流转。

## 路径 A：数据资产建设

### 步骤 1：登录

**页面：**
`/login`

**操作：**

1. 输入用户名 `DataZhang`。
2. 输入密码 `123456`。
3. 点击“登录”或按 Enter。

**页面反馈：**

- 显示“登录成功”。
- 自动进入 `/` 数据资产概览。
- 刷新后仍保持当前登录会话。

**讲解重点：**

- Demo 使用 `sessionStorage` 保存登录标识，不保存明文密码。
- 这是前端 Mock 路由保护；生产环境应由后端身份认证和权限校验负责。

**预计时间：**
20 秒

### 步骤 2：数据资产概览

**页面：**
`/`

**操作：**

1. 快速扫过六项核心统计。
2. 指向数据资产分布和最新动态。
3. 展示“核心入口与建设链路”中的 1—4 步。
4. 点击“数据接入”。

**页面反馈：**

**讲解重点：**

- 业务域按技经、勘测、电气设计、工程和标准知识组织，避免做成通用数据后台。

**预计时间：**
25 秒

### 步骤 3：查看输电线路数据接入

**页面：**
`/data-access`

**推荐稳定操作：**

1. 在“已接入数据源”中找到“广东输电线路 GIS 数据库”。
2. 点击该行操作区的“详情”眼睛图标。
3. 查看连接摘要、数据规模、同步策略和最近执行记录。
4. 关闭详情。
5. 在“接入执行记录”中打开“广东输电线路 GIS 增量同步”的“日志”图标。
6. 查看建立连接、读取数据、结构校验、元数据采集和写入原始区的步骤。

**可选扩展操作：**

1. 点击“GIS 与勘测文件”快捷卡片。
2. 选择 `Shapefile`。
3. 数据源名称填写“广东输电线路路径 GIS 演示接入”。
4. 业务领域选择“电气设计数据”。
5. 地区选择“广东省”。
6. 责任部门填写“输电工程数字化中心”。
7. 下一步保持编码“自动识别”、坐标系“CGCS2000”。
8. 点击“测试连接”，继续查看字段预览。
9. 同步方式选择“增量接入”，更新频率选择“每天”。
10. 创建接入任务，等待模拟执行完成。

**页面反馈：**

- 详情和日志展示预置数据源及任务记录。
- 可选新建流程会显示结构识别、字段预览、同步配置和成功结果。

**讲解重点：**

- 接入模块区分长期存在的“数据源配置”和每次运行的“任务实例”。
- 真实系统需要连接器、凭据托管、调度服务和原始数据区；Demo 只模拟配置与执行反馈。
- 可选新建任务不会自动在治理页生成新任务，治理阶段使用预置同名任务继续展示。

**预计时间：**
稳定操作 35 秒；含可选新建流程约 90 秒

### 步骤 4：治理输电线路 GIS 数据

**页面：**
`/data-governance`

**操作：**

1. 找到“广东输电线路 GIS 数据治理”。
2. 点击第一个治理图标，Tooltip 为“开始治理”。
3. 在工作区点击“启动治理”，等待模拟执行完成。
4. “标准化处理”保持推荐规则，点击“下一步”。
5. 在“数据质量检查”点击“全部接受”。
6. 观察高严重程度问题仍保留“待人工复核”，点击“下一步”。
7. 查看已预填的业务元数据，点击“下一步”。
8. 确认三级分类为“电气设计数据 / 线路设计 / 线路路径”，查看安全等级与标签，点击“下一步”。
9. 在治理确认中查看治理前后评分、修复数和目录路径。
10. 点击“确认完成治理”。

**页面反馈：**

- 自动执行经历加载、标准化、质量规则、元数据、分类推荐和结果生成。
- 完成后质量评分变为 97.8%，可查看治理报告或提交目录。

**讲解重点：**

- 治理不是单一清洗按钮，而是规则、质量、元数据、分类和安全分级的组合。
- “全部接受”仍保留高风险人工复核，体现自动治理与人工确认边界。
- “保存并退出”可以在同一页面会话内恢复草稿，但刷新后恢复初始 Mock。

**预计时间：**
90 秒

### 步骤 5：查看资产目录和资产详情

**页面：**
`/asset-catalog`

**推荐稳定操作：**

1. 不提交刚才的治理结果，先进入资产目录。
2. 在关键词中搜索“广东省输电线路路径”。
3. 打开“广东省输电线路路径 GIS 数据集”的“查看/编辑”图标。
4. 依次切换：
   - 基本信息
   - 元数据
   - 数据质量
   - 数据血缘
   - 分类标签
   - 价值与健康度
   - 使用与运营
5. 重点停留在“数据血缘”和“价值与健康度”。

**页面反馈：**

- 资产详情展示从来源系统、接入、治理到资产和产品的简化血缘。
- 质量、标签、安全、健康度、价值和运营指标共同构成资产详情。

**讲解重点：**

- 资产目录不只是文件列表，而是业务分类、元数据、质量、血缘、价值和运营的统一视图。
- 当前重点资产已是“已申请上架”状态，因此可直接进入下一步上架审核。

**预计时间：**
55 秒

### 步骤 6：数据产品上架

**页面：**
`/product-publish`

**操作：**

1. 在“待审核上架资产”中找到“广东省输电线路路径 GIS 数据集”。
2. 点击“开始上架”图标。
3. 查看资产来源摘要，点击“下一步”。
4. 检查预填产品名称、描述、业务域、地区、标签和责任信息，点击“下一步”。
5. 查看质量检查结果和检查详情，点击“下一步”。
6. 查看安全等级、脱敏、权限和开放方式，点击“下一步”。
7. 保持重要数据默认的指定角色与管理员审批配置，点击“下一步”。
8. 点击“产品门户预览”，关闭预览。
9. 点击“上架产品”。
10. 成功后点击“查看已上架产品”，确认产品进入列表。

**页面反馈：**

- 六步流程完成后生成 `DP-2026-xxxx` 模拟编号。
- 新产品进入当前页面“已上架产品”列表。

**讲解重点：**

- 产品不是凭空创建，而是来源于已提交上架申请的数据资产。
- 上架过程将资产封装成可服务的产品，并配置质量、安全、权限和申请策略。
- 上架页新增产品只保存在当前组件状态；门户页不会动态接收该新产品，门户展示预置的同名产品。

**预计时间：**
75 秒

**页面：**
仍在产品详情 Drawer

**操作：**

2. 确认申请人“张数据”和部门“数据管理中心”为只读。
3. 使用项目填写“粤东 500kV 输变电工程”。
4. 使用目的填写“用于线路路径比选、交叉跨越分析和工程前期踏勘”。
5. 申请方式选择“文件下载”。
6. 使用期限选择“6 个月”。
7. 数据使用范围填写“输电设计部项目组，仅限本工程设计分析”。
8. 勾选保密承诺。
9. 点击“提交申请”。
10. 在详情中点击已经变为“查看申请”的按钮。

**页面反馈：**

- 显示模拟申请编号。
- 同一产品当前会话内不能重复提交。
- 自动进入 `/workbench?tab=applications...`，切换“我的申请”并打开对应申请详情。

**讲解重点：**

- 这是本 Demo 最明确的跨页面会话联动：门户创建申请，工作台读取同一内存申请记录并定位。
- 刷新整个浏览器页面后，该新增申请会丢失。

**预计时间：**
55 秒

### 步骤 9：查看和管理我的申请

**页面：**
`/workbench?tab=applications...`

**操作：**

1. 查看自动打开的申请详情和审批节点。
2. 关闭详情，确认新申请位于“我的申请”列表。
3. 可选：对该审批中申请点击“撤回”，展示申请人侧状态控制。
4. 如需展示修改流程，选择预置的“输电线路杆塔三维模型库”已驳回申请，点击“修改申请”。
5. 查看原申请内容回填，修改使用范围后重新提交。

**页面反馈：**

- 新申请状态为“审批中”。
- 撤回后变为“已撤回”。
- 已驳回、已过期或草稿申请修改提交后变为“审批中”。

**讲解重点：**

- 使用者侧聚焦收藏、申请详情、撤回和整改后重新提交。
- 为保证后续审批演示稳定，建议不要撤回刚提交的主线申请。

**预计时间：**
40 秒

### 步骤 10：管理员审批

**页面：**

**操作：**

2. 选择产品为“广东省输电线路路径 GIS 数据集”的待审批记录。
3. 点击审批图标。

**页面反馈：**

- 待审批数量减少。
- 本月完成审批数量增加。
- 生成一条本地审批历史记录。

**讲解重点：**

- 管理员侧根据申请人、用途、期限、安全等级和最小化原则进行风险判断。
- 当前待审批队列是独立预置 Mock，不是步骤 8 刚提交的同一条申请；生产系统应由后端审批流统一驱动双方状态。

**预计时间：**
60 秒

## 备用路径

- **登录状态异常**：在开发者工具 Application → Session Storage 中删除 `data-asset-demo-auth`，刷新 `/login`。
- **本地状态被改乱**：刷新页面恢复该模块初始 Mock；若需重置所有内存共享状态，刷新浏览器标签页后重新登录。
- **治理流程时间不足**：打开预置已完成任务“深圳电网规划项目数据治理”，直接查看结果；主线讲解仍使用广东输电线路 GIS。
- **重点资产未出现在待审核列表**：刷新页面；初始 Mock 中“广东省输电线路路径 GIS 数据集”应处于已申请上架且未列入初始已上架产品的状态。
- **门户申请已提交过**：刷新门户后重新搜索，或换用“粤港澳大湾区数字高程模型”。
- **审批记录已处理**：刷新工作台恢复初始待审批 Mock。

## 不建议现场点击的功能

- 不建议先对“广东省输电线路路径 GIS 数据集”执行取消上架申请，否则会影响待审核上架演示。
- 不建议在上架演示前下架已有产品，避免待审核列表内容变化。
- 不建议演示治理异常资产“退回”后立刻跨页面追踪；该联动依赖内存状态，刷新会重置。
- 不建议在主线申请后立即撤回，否则不利于解释申请处理中状态。
- 不建议将“新建接入任务”描述为会自动生成治理任务；代码未实现该动态共享。
- 不建议声称新治理资产会自动进入上架工作区；上架页主要读取初始资产及其状态覆盖。
- 不建议声称新上架产品会动态加入门户列表；门户使用独立预置产品 Mock。
- 不建议声称门户提交的申请会进入“待我审批”；它只会进入当前用户的“我的申请”。
- 下载、在线查看、API 凭证等入口只提供提示或展示，不应作为主线。

## 演示风险检查

| 风险 | 当前表现 | 演示建议 |
| --- | --- | --- |
| 刷新丢失业务状态 | 收藏、治理草稿、上架结果、审批结果等多为本地状态 | 演示前刷新，开始后避免无必要刷新 |
| 登录状态与业务状态范围不同 | 登录保存在 sessionStorage；业务状态多在内存 | 刷新可保持登录，但可能重置业务操作 |
